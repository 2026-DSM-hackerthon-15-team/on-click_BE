# 목적
현재 너는 해커톤에서의 서버를 개발하는 AI이다.
우리는 여러 주제중 소상공인들이 운영하는 가게를 부양시키는 아이디어를 선택하였다.
## 프로젝트의 의의
프로젝트의 목적은 이러하다.
**소상공인들을 돕기위해, 그들이 어려워하는 컨설팅 / 마케팅 쪽을 AI로 도울수 있는 AI 비서를 만들자**
제일 중점으로 들고갈 내용은 컨설팅이다.
## 프젝 설계
포스기를 연결하여 실행하여 편의성을 높이고, 더욱 더 쉽게 도울 수 있다는 가정하에 진행한다.(실제로 포스기를 연동하지 않고 포스기처럼 쓸 수 있는 API를 개발한다.)
그리고 AI서버를 함께 개발하며 진행하며, 이 프로젝트에서는 관련된 부분을 처리하지 않고 해당 서버에 API 호출을 하는 식으로 처리한다. 일단 서버가 준비가 된 상황이 아니니 mockData로 처리할 예정이다.
또한 위와 같은 경우는 직접 명세한다. (ex. 마감전 가격 예측(AI서버 책임))
또한 모든 프론트의 요청은 백엔드를 통해 진행된다.
### 상세 기능
1. 대시보드
- 오늘 매출, 오늘 주문 건, 방문자 수를 조회한다.
- 시간대별 매출, 시간대별 방문자를 표시해준다.(오늘기준)
- 전에 있던 데이터를 통해 마감매출(AI 서버 역할), 내일 방문자를 예측한다(AI 서버 역할).
2. 채팅
- 기본적으로 채팅으로 작동한다. 백엔드는 AI 서버가 답변 생성에 필요한 매장·판매·대화 정보를 요청에 포함한다.
- 인스타 홍보/컨설팅의 경우에는 직접 채팅을 통해 알려주기보다는 해당 링크로 이동시킨다. (AI 서버)
- 백엔드 서버에 요청이 들어오면 AI 서버에 생성 작업을 제출하고, AI 서버는 완료 결과를 백엔드 callback API로 전달한다. 백엔드는 이를 저장하며 프론트는 GET 요청으로 polling한다.
3. 인스타 홍보
- 인스타 홍보 게시글 내용을 AI서버에서 post요청으로 생성해주면, 그걸 사용자가 리스트를 조회 후 승인한다. 추가적으로 필요시 내용 업데이트 가능하다.
- Meta OAuth 대신 매장별 Instagram 로그인 계정(ID·비밀번호)을 저장하고 브라우저 MCP 자동화에서 사용한다.
4. 컨설팅
- 매일 저녁마다 한번씩 기존 판매 내역 및 여러 정보들을 취합하여 컨설팅을 AI서버에서 제작하고 이를 post요청으로 백엔드 서버에 저장한다. 이를 조회 가능하다. 리스트, 상세 조회가 가능하여야한다.
- 컨설팅은 거의 컨설팅 내용의 content가 대부분이고, 날짜, 제목 등이 남은 속성을 차지한다.
5. 회원가입 / 로그인 / 마이페이지
- 회원가입 시에 아이디 / 비번 / 지점 등록이 가능하다. 이부분은 이후에 마이페이지에서 수정이 가능하며, 이를 통해 여러 지점을 관리 가능하다.
- 또한 지점마다 모두 따로 관리되어야한다.
- 그리고 계정의 정보도 관리 가능하다.
6. 채팅방
- 채팅방은 여러 채팅방들을 저장 가능하다. 채팅방마다 채팅이 여러개 존재하고, 속성으로 AI가 보낸건지, 내가 보낸건지가 나뉜다.
- 채팅방 삭제시 채팅들도 삭제된다.

1. gradle로 진행할것
2. 파일 구조는 크게 domain, global, common으로 나눈 후 그 안에서 진행할것. domain은 특정 domain에 대한것(user, book..) global은 전체에서 사용하는 파일 MessageResponse, config 등
   common은 전체적인 남는거
3. ./gradlew build를 시간날때마다 진행할것
4. 기능이 구현될때마다 git commit을 진행할것
